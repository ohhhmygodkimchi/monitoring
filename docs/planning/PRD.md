# 제품 요구사항 정의서 (PRD)

## 1. 프로젝트 개요 (Overview)
- **프로젝트 명:** 온프레미스 K8s 모니터링 파이프라인 구축
- **작성일:** 2026-03-17
- **목표:** 온프레미스 Kubernetes(K8s) 환경에서 실행 중인 서비스들의 장애 포인트를 신속히 파악하고 대응하기 위한 종합 모니터링 시스템 구축.

## 2. 배경 및 목적 (Background & Objectives)
- **배경 (Pain Point):** 현재 시스템에 모니터링 툴이 부재하여, 서비스 지연이나 장애 발생 시 원인 컴포넌트(서버 자원, 애플리케이션 로직, DB 병목 등)를 특정하거나 런타임 에러를 실시간으로 파악하기 어려움. 이로 인해 사후 대응(수동 로그 확인 등)에 오랜 시간이 소요됨.
- **목표 (Goal):** 인프라, 애플리케이션(APM), 에러 트래킹, 통합 로그 수집에 이르는 전체 모니터링 파이프라인을 구축하고 시각화 대시보드 및 실시간 알림 채널을 마련하여 **MTTR(평균 복구 시간)**을 단축하고 시스템 안정성을 확보함.

## 3. 핵심 기능 요구사항 (Core Features)

### 3.1 인프라 모니터링 (Infrastructure Monitoring)
- K8s 클러스터 내 노드 및 Pod의 상태 (CPU, 메모리, 네트워크, 디스크 I/O 등) 실시간 수집
- 자원 임계점 도달 시 알림 트리거

### 3.2 애플리케이션 성능 모니터링 (APM - Application Performance Monitoring)
- 서비스 간 분산 트레이싱(Distributed Tracing) 지원으로 병목 구간 탐지
- API 응답 속도, DB 쿼리 실행 시간 등 코드 레벨 지표 측정

### 3.3 에러 트래킹 (Error Tracking)
- 사용자 및 서버 런타임 환경에서 발생하는 에러 자동 수집
- 에러 발생 맥락(Call stack, 사용자 환경 정보 등) 제공 및 중복 에러 그룹화

### 3.4 통합 로그 관리 (Centralized Logging)
- 흩어진 K8s Pod들의 로그를 중앙 서버로 실시간 수집 및 인덱싱
- 특정 시점(장애 발생 시간대 등) 전후의 로그 검색 및 상관관계 분석 기능

### 3.5 알림(Alerting) 및 대시보드(Dashboard)
- 메트릭(수치형 데이터) 시각화 통합 대시보드 제공
- 심각한 오류 발생 시 또는 자원 리미트 초과 시 외부 채널로 즉각 알림 발송

## 4. 시스템 요구사항 (System Requirements)
- **실행 환경:** 온프레미스 Kubernetes (K8s) 클러스터
- **배포 방식:** **Helm**을 이용한 패키지 배포. 모든 모니터링 컴포넌트는 Helm의 `values.yaml` 파일을 템플릿화하여 환경에 맞게 수정한 후 배포(`helm install/upgrade`)하는 방식을 엄격하게 준수함.

## 5. 단계별 검토 사항 (Next Steps / TBD)
*비고: 사용자와의 추가 논의가 필요한 사항들*
1. **현재 최우선 순위 도출:** 4가지 모니터링 항목 중 가장 시급하게 도입해야 하는 병목 지점 확인.
