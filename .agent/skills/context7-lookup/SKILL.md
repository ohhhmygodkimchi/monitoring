# Context7 기술 문서 검색 스킬

## 사용 시점
- 기술 스택을 선정하거나 라이브러리를 비교할 때
- 구현 중 오류가 발생하여 공식 문서를 참조해야 할 때
- 테스트 중 버그 원인 분석을 위해 라이브러리 동작을 확인할 때
- 특정 프레임워크/라이브러리의 최신 API나 사용법을 확인할 때

## 사용 방법

### 1단계: 라이브러리 ID 검색
- `resolve-library-id` 도구를 호출하여 Context7 호환 라이브러리 ID를 획득한다.
- 검색 결과에서 **Source Reputation**(High 우선), **Code Snippets 수**, **Benchmark Score**를 기준으로 가장 적합한 라이브러리를 선택한다.

### 2단계: 문서 질의
- `query-docs` 도구에 획득한 라이브러리 ID와 구체적인 질문을 전달한다.
- **좋은 질문 예시**: "How to set up authentication with JWT in Express.js"
- **나쁜 질문 예시**: "auth" (너무 모호함)

### 3단계: 결과 적용
- 검색 결과를 바탕으로 코드를 수정하거나 기술 결정을 내린다.
- 사용한 라이브러리 ID와 참조한 문서 내용을 관련 보고서에 기록한다.

## Context7 도구 미감지 시 행동 요령
1. `context7` 관련 도구가 사용 가능한 목록(Tools)에 보이지 않는다면, 즉시 사용자에게 **"Context7 서비스를 사용할 수 없습니다. 활성화 여부를 확인해 주시겠습니까?"**라고 질문한다.
2. 사용자가 MCP 사용이 불가능하다고 할 경우, **일반 웹 검색(`search_web`)** 또는 **내부 지식**을 사용하여 대체한다.

## 주의사항
- `query-docs`는 **질문당 최대 3회**까지만 호출한다. 3회 내 원하는 정보를 찾지 못하면 가진 정보로 판단한다.
- 선정된 기술의 **Context7 라이브러리 ID**는 반드시 `docs/planning/TECH_STACK.md`에 기록하여, 이후 단계에서 동일한 ID로 바로 문서를 조회할 수 있게 한다.
