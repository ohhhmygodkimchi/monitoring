# 단계 전환 워크플로우 (Stage Transition)

> 현재 단계를 마치고 다음 단계로 넘어갈 때 아래 절차를 **반드시** 수행합니다.

## 절차

1. **체크리스트 검증**
   - `.agent/PROJECT_STATUS.md`에서 **현재 단계의 체크리스트**를 확인한다.
   - 모든 항목이 `[x]`(완료)인지 검증한다.
   - 미완료 항목이 있으면:
     ```
     ⚠️ 미완료 항목 발견
     - [ ] [미완료 항목 내용]
     해당 항목을 건너뛰고 진행하시겠습니까?
     ```
     사용자 승인을 받은 후에만 계속한다.

2. **현재 단계 체크리스트 완료 처리**
   - 승인된 모든 항목을 `[x]`로 업데이트한다.

3. **현재 단계 정보 갱신**
   - `PROJECT_STATUS.md`의 **현재 단계(Current Stage)**를 다음 단계로 업데이트한다.
   - 예: `**[1. 기획 단계 (PM)]**` → `**[2. 설계 단계 (PL)]**`

4. **Next Action Hook 작성**
   - 다음 에이전트(또는 다음 세션의 자신)가 **즉시 작업을 시작할 수 있도록** 구체적인 지침을 작성한다.
   - 반드시 포함할 내용:
     - 다음에 읽어야 할 역할 파일 (`NN_ROLE_TASK.md`)
     - 참조해야 할 핵심 문서 목록
     - 우선 수행할 작업 요약

5. **Change Log 기록**
   - `PROJECT_STATUS.md`의 **변경 로그** 테이블에 아래 형식으로 추가한다:
   ```
   | YYYY-MM-DD | [완료 단계] → [다음 단계] | [수행 내용 요약] | [비고] |
   ```

6. **사용자에게 전환 보고**
   ```
   ✅ 단계 전환 완료
   - 완료 단계: [이전 단계명]
   - 다음 단계: [새 단계명]
   - 권장 모델: [다음 단계 권장 모델]
   - 즉시 시작 가능한 작업: [Next Action Hook 요약]
   ```
